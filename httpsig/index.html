Notes
  1. This tool uses EcmaScript v9, and webcrypto; it will run only on modern, current browsers.
  2. For generating and verifying HTTP Signature, this page supports rsa-sha256 or hmac-sha256.
  3. To generate:
    1. paste in the headers in the upper left textarea.
    2. select an algorithm in the dropdown.
    3. specify an appropriate key.
    4. click the right arrow.
  4. To verify:
    1. paste in the headers in the upper left textarea.
    2. paste in your full Signature header in the lower right textarea.
    3. click the checkmark icon.
  5. On initial page load, the browser will generate a new RSA keypair and insert it into the appropriate boxes. It also loads some contrived fixed headers. You can replace either or both with your own values.
  6. You can generate a new asymmetric keypair by clicking the key button.
  7. When specifying the symmetric key, you need at least 32 bytes of key material for hmac-sha256, whether signing or verifying. The PBKDF2 will generate keys of the appropriate size. If you choose a different coding system, you need to ensure minimum key size yourself.
  8. Information that you paste here, including signatures and keys, whether secret, private or public, never leaves your browser. Information from the decoding also stays in the browser. Nothing gets posted to a server. Still, you should take care not to use this site for private information.
  9. This page does not explicitly handle the special values like (target-request), (created), or (expires). I may extend the page in the future to support some or all of those.
