Notes:
  1. This page uses the WebCrypto library, for the hash and HMACs computation.
  2. Any data you enter on this page stays in the browser, and is never sent to any third-party site. The hash computation is performed in your browser. If you specify a key for HMAC, the key is used only for HMAC calculation, and never leaves your browser.
  3. The results shown by this page are different encodings that represent the same value.
  4. Base16 encoding is sometimes called "Hex encoding".

  6. Just a reminder, base64-encoded values are case sensitive, base16-encoded results are not. This is characteristic of the encodings, not special to this page, or to hash functions or HMACs.
  7. For message content, whitespace in the content is significant. This page converts line-breaks to newline (\n). It's not possible to include a carriage-return (CR) in the text value, because the logic on this page strips them. Trailing newlines will not be stripped. Also good to remember: the hash for 'abc\n' is completely different from the hash for 'abc'. The hash for 'abc\r\n' is completely different from the hash for 'abc\n'.
  8. Unicode is ok. The calculator will encode the string as UTF-8 bytes before computing the checksum. UTF-8 ought to be good enough for anybody. :)
